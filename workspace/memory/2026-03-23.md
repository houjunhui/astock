## A股超短系统 v3.1 Bug修复（4个）

### Bug 1：概率标定表数据被覆盖（INSERT OR REPLACE）
- **问题**：`INSERT OR REPLACE` 每天会删除旧行再插入新行，导致累计数据被覆盖
- **修复**：改为 `DELETE FROM calibration` + 从头重新计算所有历史样本的累计值

### Bug 2：结局显示None（游标误用）
- **问题**：`rows = conn.execute('SELECT * ...').fetchall()` 后，用 `for row in rows` 循环中执行 `UPDATE`，然后又用同一个 `rows` 变量（游标已耗尽）显示数据，导致全部显示 None
- **修复**：UPDATE后重新执行查询得到 `rows2`，SELECT明确包含所有必要字段

### Bug 3：预测均值显示525%（重复乘100）
- **问题**：`avg_pred` 存储的已经是百分数（5.25 = 5.25%），但格式化时又乘了100
- **原因**：`update_calibration` 存的是 `jb_prob` 的原始值（5.25），`get_calibration_report` 里 `avg_pred = pred_sum/n`（5.25），然后 `f"{avg_pred*100:.1f}"` 变成了525
- **修复**：直接 `f"{avg_pred:.2f}"`，不乘100

### Bug 4：偏差列单位混乱
- **问题**：`bias = actual - avg_pred`（actual是小数0.0，avg_pred是百分数5.25），然后bias被乘100打印成-525%
- **修复**：`bias_pct = actual*100 - avg_pred`，统一为百分点差

### 修复后数据验证
